Project & Video

The idea for this project started out in 2010 when I bought my first Arduino. I would do some very simple things with it but had always wanted to build a web interface for it so that others could control as well. I had experimented with web design for a number of years before this and had enjoyed creating wordpress themes for friends and clients. My first approach to combining my web skills with my Arduino skills was to create a php page hosted on the Arduino's SD card and forward incoming traffic on my modem to the Arduino shields ethernet port. For security reasons, this is not a very wise idea as it leaves your home network open to the world...well, sort of.

It wasn't until a few years ago that I learned about NodeJs and the Johnny-Five framework that allowed me to take another crack at my original idea. This time, instead of using php, I would be using javascript and html5. I would still have a server set up at my home to control the Arduino, but then I would also have the actual website (that you are now viewing) hosted on Amazon's cloud services for the webpage. By doing it this way, I would not leave my home network open to potential intruders.

Every time an Arduino sensor changes value (eg. motion sensor), it sends that value to a Redis server in the cloud. The same principle applies to the input controls on the website (eg. LED color picker). When you change a value (color), it sends the message to the Redis server in the cloud and then to my Arduino microprocessor.

If you are having trouble grasping this, no worries, it took me months to figure it all out. Please watch the video below for more information.

Node.js

Node.js is at the heart of this website. It is an open source, cross-platform server environment for networked applications. In other words, it is a number of tools that allow you to build applications in JavaScript. Within Node.js, I've used a framework called Express to build this application, specifically for the web. Another major JavaScript framework I've used within this project is Johnny-Five. This framework made it very easy to control the Arduino. Lastly, another framework, called Socket.io was used to create real-time event-based communication between the website and the Arduino.

Build toos like Gulp (and Grunt) are any web developers best friend as it allows you to do many things whether it be preprocess your LESS code to css format, minify your javascript/html as you save your work or even more advanced features such as Browsersync so that if you scroll your mouse, and you have multiple screens (i.e: phone, tablet, and monitor) to all scroll at the same time. It's pretty magical and it saves you a lot of time in development! Below are some links to help you get started if you are interested in learning more:

Amazon AWS

Amazon AWS is a collection of remote computing services, also called web services, that make up a cloud computing platform offered by Amazon.com. Because of this, Amazon is Awesome! They allow you to run a virtual environment (windows or linux) in the cloud that you can use to do many different things with. A general example is using EC2 to store your website, VPC to organize your Virtual Private Networking Inferstructure, and S3 to store all of your media (photos/videos/pdf's etc). They even have a domain system that you can now buy your domains from (ex bradlannon.com). This is just the tip of the iceberg. Because of these new advances in Cloud computing, I no longer think 'bookstore' when I think of Amazon.
